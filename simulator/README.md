# 🧪 ESP32 Hardware Simulator

شبیه‌ساز سخت‌افزار ESP32 برای تست داشبورد **بدون دستگاه فیزیکی**.

این شبیه‌ساز دقیقاً همان HTTP API سخت‌افزار واقعی (`esp32/main.py`) را ارائه می‌دهد،
پس می‌توانید کل پایپ‌لاین (آلارم، صدا، آستانه‌ها، real-time) را تست کنید و بعداً بدون
هیچ تغییری در کد، به سخت‌افزار واقعی سوییچ کنید.

## اجرا

```bash
python simulator/esp32_sim.py --scenario demo
```

پیش‌فرض روی پورت **8080** بالا می‌آید: `http://127.0.0.1:8080`

### آرگومان‌ها
| آرگومان | پیش‌فرض | توضیح |
|---------|---------|-------|
| `--scenario` | `demo` | سناریو: `demo`, `gas_leak`, `fire`, `intrusion`, `normal` |
| `--port` | `8080` | پورت |
| `--host` | `0.0.0.0` | هاست |
| `--warmup` | `30` | ثانیه‌های warmup سنسور MQ9 (برای تست سریع `--warmup 3`) |
| `--interval` | `2` | فاصله‌ی خواندن سنسور (پیش‌فرض دستگاه ۲ ثانیه است) |

همه با env هم قابل تنظیم‌اند: `SIM_SCENARIO`, `SIM_PORT`, `SIM_WARMUP_SEC`, `SIM_READ_INTERVAL`.

## سناریوها
- **demo** — چرخه‌ای از همه‌ی آلارم‌ها (گاز → دما → شعله → نفوذ) با فاصله‌های عادی. بی‌نهایت تکرار می‌شود. برای دیدن کارکرد کامل داشبورد ایده‌آل است.
- **gas_leak** — بعد از چند ثانیه نشت گاز پایدار (mq9 از ۲۰۰۰ رد می‌شود).
- **fire** — شعله + افزایش دما.
- **intrusion** — حرکت + باز شدن درب.
- **normal** — همیشه آرام (تست حالت بدون آلارم).

## Endpointها (آینه‌ی سخت‌افزار)
| Method | Path | توضیح |
|--------|------|-------|
| GET | `/` | صفحه وضعیت (برای چک آنلاین بودن) |
| GET | `/data` | داده زنده سنسورها (JSON دقیقاً مثل دستگاه) |
| GET | `/config` | آستانه‌ها و تنظیمات |
| POST | `/config` | به‌روزرسانی آستانه‌ها (فقط کلیدهای مجاز) |
| GET | `/diag` | صفحه دیاگنوز |

## اتصال داشبورد به شبیه‌ساز
در فایل `.env`:
```env
ESP32_BASE_URL=http://127.0.0.1:8080
```
سپس `python app.py` را اجرا کنید.

## سوییچ به سخت‌افزار واقعی
کافی است `ESP32_BASE_URL` را به IP دستگاه برگردانید (مثلاً `http://192.168.43.219`).
هیچ تغییر کد دیگری لازم نیست — همان منطق آستانه/آلارم روی خود سخت‌افزار اجرا می‌شود.

## دقت شبیه‌سازی
- مقادیر آنالوگ (گاز/دما) با شیب نرم و نویز واقعی‌نما تغییر می‌کنند.
- شعله از همان ماشین حالت hysteresis دستگاه عبور می‌کند (۴ سیکل LOW برای روشن، ۱۰ سیکل HIGH برای خاموش).
- منطق آلارم و آستانه‌ها عیناً مثل `esp32/main.py` است.
- `warmup_done` و `alarm_reasons` دقیقاً مطابق دستگاه برگردانده می‌شوند.
